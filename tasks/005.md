# Описание рисков реализации (бизнес и технические)

* Бизнес
1. Негативные отзывы у приложения. Репутационные риски.
2. Слабые поведенческие факторы у приложения, несоответствие ожиданиям целевой аудитории.
* Технические
1. Сценарий: «Тренировка в диком месте»
​Пользователь начинает бег в горах (нет сети).
​Приложение пишет телеметрию в локальную БД на телефоне.
​При появлении сети происходит синхронизация с Activity Service.
​Риск: Конфликт версий данных или потеря данных при сбое телефона до синхронизации.
​Решение: Локальный append-only лог, retry-политики.
2. Сценарий: «Масштабное соревнование»
​Тысячи пользователей одновременно стартуют.
​Резкий скачок нагрузки на запись телеметрии.
​Риск: Отказ БД из-за нагрузки на запись.
​Решение: Использование Sharding для БД телеметрии; очередь сообщений (Kafka) для сглаживания пиков (Backpressure).
